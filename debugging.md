# デバッグの進め方

[< ガイド目次に戻る](README.md)

---

バグに遭遇したとき、Claude Codeは強力なデバッグパートナーになります。

## 基本パターン：エラーを渡して調査させる

```
あなた: このエラーが出る。原因を調べて修正して。

        TypeError: Cannot read properties of undefined (reading 'map')
        at UserList (src/components/UserList.tsx:15:23)

Claude: エラーの発生箇所を確認します...

        src/components/UserList.tsx:15 で users.map() を呼んでいますが、
        usersが undefined になるケースがあります。

        原因: useEffect内のAPI呼び出しが完了する前にレンダリングが走っている
        修正: users に初期値 [] を設定します。

        （修正を実施 → テスト実行 → ✅ 通過）
```

## パターン1：Plan Modeで原因調査

原因がわからない複雑なバグは、**Plan Modeで調査してから修正**するのが安全です。

```
# Shift+Tab でPlan Modeに入る

あなた: ログイン後にセッションが維持されないバグがある。
        以下のファイルが関係しそう：
        - src/auth/session.ts
        - src/middleware/auth.ts
        原因を調べて。修正はまだしないで。

Claude: （コードベースを読み取り専用で調査）
        原因を特定しました。session.ts のトークンリフレッシュ処理で...

# 原因が分かったら Shift+Tab で通常モードに戻して修正
```

## パターン2：ログを渡す

```bash
# エラーログをパイプで直接渡す
cat error.log | claude "このエラーログを分析して、原因と修正方法を教えて"

# Docker環境のログ
docker compose logs api-server --tail 50 | claude "このログからエラーの原因を特定して"
```

## パターン3：再現テストから書く

バグの修正で最も確実なのは、**先に再現テストを書く**方法です。

```
あなた: ユーザー名が空文字のときに500エラーが返るバグがある。
        まず、このバグを再現するテストを書いて。
        テストが失敗することを確認してから、修正して。

Claude: 再現テストを作成しました。
        ❌ 1/1 failed（バグ再現を確認）

        修正を適用します...
        ✅ 1/1 passed（修正完了）
```

## デバッグ時の指示のコツ

```
❌ 「動かない」
✅ 「画面に何も表示されない。コンソールに以下のエラーが出る：[エラー内容]」

❌ 「遅い」
✅ 「/api/users のレスポンスが5秒以上かかる。DBクエリが原因だと思う」

❌ 「前は動いてた」
✅ 「先週のコミット abc123 までは動いていた。git log を見て、
    何が変わったか調べて」
```

ポイント：**症状・発生箇所・再現手順**を具体的に伝える。
