## Claude Codeとは

Claude Codeは、ターミナルに常駐するAIエージェントです。単なるチャットボットではなく、以下を**自律的に**実行できます。

- ファイルの読み書き・編集
- コマンド実行（ビルド、テスト、git操作など）
- Web検索による最新情報の取得
- プロジェクト全体の構造理解

```bash
# 起動
claude

# プロジェクトディレクトリで起動するのが基本
cd my-project && claude
```

## 権限モードと承認（重要）

Claude Codeはファイル編集やコマンド実行など**システムに影響を与える操作**を行います。実行前に承認を求められるのは安全装置です。**安易に全承認にしないでください。**

### 3つの操作モード

`Shift + Tab` を押すたびにモードが切り替わります：

```
通常モード → Auto-Acceptモード → Plan Mode → 通常モード → ...
```

| モード | 表示 | 動作 | 安全性 |
|--------|------|------|--------|
| **通常モード** | （表示なし） | ファイル編集・コマンド実行のたびに承認を求める | 最も安全 |
| **Auto-Accept** | `⏵⏵ accept edits on` | ファイル編集を自動承認。Bashコマンドは引き続き確認される | 中程度 |
| **Plan Mode** | `⏸ plan mode on` | 読み取り専用。一切の変更を行わない | 完全に安全 |

### 承認プロンプトの読み方

通常モードでは、Claudeが操作を行う前に確認が表示されます：

```
Claude wants to edit src/app.ts
Allow? [y/n/a]

  y = この1回だけ許可
  n = 拒否
  a = このセッション中、同種の操作を全て許可
```

**Bashコマンドの場合は特に注意**：

```
Claude wants to run: rm -rf dist/
Allow? [y/n/a]
```

実行されるコマンドを**必ず読んでから**承認してください。`rm`、`git push --force`、`DROP TABLE` などの破壊的な操作が含まれていないか確認しましょう。

### 事故を防ぐためのルール

| ルール | 理由 |
|--------|------|
| **最初は通常モードで使う** | 慣れるまではClaude が何をしようとしているか毎回確認する |
| **Auto-Acceptは信頼できるタスクだけ** | 「テスト書いて」「リファクタして」など、影響範囲が明確なとき |
| **Bashコマンドは毎回読む** | ファイル削除、パッケージの追加/削除、git操作は特に注意 |
| **迷ったらPlan Mode** | 読み取り専用なので何も壊れない。調査・計画フェーズに最適 |
| **`--dangerously-skip-permissions` は使わない** | 全ての安全装置をバイパスするフラグ。CI以外では使用禁止 |

### 実際に起きうる事故の例

```
❌ Auto-Acceptで「プロジェクトを整理して」と依頼
   → Claudeが不要と判断したファイルを大量に削除

❌ Bashコマンドを読まずに承認
   → git push --force で他のメンバーの変更を上書き

❌ 「環境をクリーンにして」と曖昧に指示
   → node_modules/ だけでなく .env も削除される
```

> **鉄則：Claudeは優秀だが、破壊的な操作の最終判断は必ず人間が行う。**
> 承認プロンプトは「面倒な確認」ではなく「あなたを守るセーフティネット」です。

## 基本操作

### よく使うコマンド

| コマンド | 説明 |
|----------|------|
| `/model` | モデルを切り替え |
| `/fast` | Fast Mode（同じモデルで高速出力） |
| `/help` | ヘルプ表示 |
| `/clear` | コンテキストをクリア |
| `/compact` | 会話を要約してコンテキストを節約 |
| `Esc` | 実行中のタスクを中断 |

### 効果的な指示の出し方

```
❌ 悪い例: 「ログイン機能を作って」
✅ 良い例: 「Express + JWTでログインAPIを作って。
            POST /api/auth/login で email と password を受け取り、
            JWTトークンを返す。Prismaでusersテーブルを参照」

❌ 悪い例: 「バグを直して」
✅ 良い例: 「src/controllers/user.ts の getUser関数で、
            ユーザーが見つからない時に500エラーが返る。
            404を返すように修正して」
```

ポイント：**何を・どこで・どうしたいか**を具体的に。
