## AI・LLM関連

| 用語 | 読み方 | 意味 |
|------|--------|------|
| **LLM** | エルエルエム | Large Language Model（大規模言語モデル）。ChatGPTやClaudeの中核技術。大量のテキストデータから学習し、人間のような文章を生成する |
| **トークン** | — | AIが文章を処理する最小単位。日本語1文字 ≒ 1〜3トークン、英単語1語 ≒ 1トークン程度。コストや使用量の計算に使われる |
| **コンテキストウィンドウ** | — | AIが一度に覚えていられる情報量の上限。会話、読んだファイル、コマンド出力が全てここに蓄積される。満杯に近づくとAIの精度が下がる |
| **ハルシネーション** | — | AIが事実と異なる内容をもっともらしく生成すること。「嘘をつく」とも言われる。テストやビルドで検証することで防げる |
| **プロンプト** | — | AIへの指示・質問文のこと。「プロンプトが上手い」＝「AIへの指示が的確」 |
| **推論（inference）** | すいろん | AIがプロンプトを受け取ってから回答を生成するプロセス。「推論コスト」＝AIを動かすのにかかるお金 |
| **ファインチューニング** | — | 既存のAIモデルを特定の用途向けに追加学習させること。Claude Codeでは行わない（プロンプトで制御する） |

## Claude Code関連

| 用語 | 読み方 | 意味 |
|------|--------|------|
| **Claude Code** | クロード コード | Anthropic社が提供するターミナルベースのAIコーディングエージェント。このガイドの主役 |
| **エージェント（Agent）** | — | 自律的にタスクを遂行するAI。Claude Codeは「ファイルを読む→考える→編集する→テストする」を自分で判断して実行できる |
| **サブエージェント** | — | Claude Codeの中で動く専門特化したAIアシスタント。`.claude/agents/` にMarkdownで定義する |
| **スキル（Skill）** | — | `/スキル名` で呼び出せるテンプレート化された指示。`.claude/skills/<name>/SKILL.md` に定義する |
| **CLAUDE.md** | — | プロジェクトのルートに置く指示書。Claude Codeが毎回自動的に読み込む。プロジェクトのルール、技術スタック、規約を書く |
| **Plan Mode** | プランモード | 読み取り専用の探索・設計モード。コードベースを調査して計画を立てるが、ファイルの変更は行わない。`Shift+Tab` で切り替え |
| **Auto-Accept** | オートアクセプト | ファイル編集の承認を自動化するモード。Bashコマンドは引き続き確認される。`Shift+Tab` で切り替え |
| **MCP** | エムシーピー | Model Context Protocol。Claude Codeに外部ツールを接続する仕組み。「Claude CodeのUSB-Cポート」 |
| **Hooks** | フックス | Claude Codeのツール実行前後に自動でシェルコマンドを実行する仕組み。例：ファイル編集後に自動フォーマット |
| **コンテキスト** | — | Claude Codeとの会話の蓄積。「コンテキストが一杯」＝これ以上情報を覚えられない状態 |
| **/clear** | クリア | コンテキストを完全にリセットするコマンド。タスクが変わるときに使う |
| **/compact** | コンパクト | 会話を要約してコンテキストを圧縮するコマンド。セッションを延命させたいときに使う |
| **チェックポイント** | — | Claude Codeが操作を行うたびに自動保存される復元ポイント。`Esc+Esc` で任意の地点に巻き戻せる |

## モデル関連

| 用語 | 読み方 | 意味 |
|------|--------|------|
| **Opus** | オーパス | Claudeの最上位モデル。最高の推論能力を持つが、コストが高い |
| **Sonnet** | ソネット | Claudeの中位モデル。4.6でOpusに肉薄するコーディング性能を獲得。コスパ最強 |
| **Haiku** | ハイク | Claudeの最軽量モデル。最速・最安。簡単なタスク向き |
| **SWE-bench** | スウィーベンチ | ソフトウェアエンジニアリングのベンチマーク。実際のGitHubのissueをAIが解決できるかで測定する。コーディング能力の指標 |

## 開発関連

| 用語 | 読み方 | 意味 |
|------|--------|------|
| **TDD** | ティーディーディー | Test-Driven Development（テスト駆動開発）。先にテストを書き、そのテストを通す実装を後から書く手法。Claude Codeとの相性が良い |
| **E2Eテスト** | エンドツーエンド | End-to-Endテスト。実際のブラウザを動かしてユーザー操作を再現するテスト。Playwright等を使う |
| **Playwright** | プレイライト | Microsoftが開発したブラウザ自動操作ツール。E2Eテストに使う。Claude Codeが書いて実行できる |
| **CI/CD** | シーアイ シーディー | Continuous Integration / Continuous Delivery。コードのビルド・テスト・デプロイを自動化する仕組み |
| **MVP** | エムブイピー | Minimum Viable Product（最小限の実用製品）。最小限の機能でまず世に出し、フィードバックを得る戦略 |
| **PoC** | ピーオーシー | Proof of Concept（概念実証）。アイデアが技術的に実現可能かを確かめるための試作 |
| **Conventional Commits** | — | コミットメッセージの書き方規約。`feat:`, `fix:`, `refactor:` 等のプレフィックスで変更の種類を示す |
| **Clean Architecture** | — | ソフトウェア設計パターンの一つ。ビジネスロジック（domain）をフレームワークやDBから独立させる設計 |
| **ORM** | オーアールエム | Object-Relational Mapping。プログラムのオブジェクトとDBのテーブルを対応させる技術。Prisma, TypeORM等 |
| **WSL** | ダブリューエスエル | Windows Subsystem for Linux。Windows上でLinuxを動かす仕組み。Claude Codeを使うWindows環境で推奨 |
