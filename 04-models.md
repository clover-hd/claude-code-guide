## モデルの違いと選び方

Claude Codeでは `/model` コマンドでモデルを切り替えられます。

### モデル一覧（2026年3月時点）

| モデル | モデルID | 特徴 | 入力/出力コスト |
|--------|----------|------|----------------|
| **Opus 4.6** | `claude-opus-4-6` | 最高性能。複雑な推論・大規模設計に最適 | $15 / $75 per 1M tokens |
| **Sonnet 4.6** | `claude-sonnet-4-6` | **コスパ最強。コーディングでOpusに肉薄** | $3 / $15 per 1M tokens |
| **Haiku 4.5** | `claude-haiku-4-5` | 最速・最安。簡単なタスク向き | 最安 |

### どう使い分ける？

```
日常的なコーディング、バグ修正、機能追加
  → Sonnet 4.6（これで十分。Opusの1/5の価格）

大規模なアーキテクチャ設計、複雑なリファクタリング
  → Opus 4.6（深い推論が必要な場面で真価を発揮）

簡単な質問、ファイル検索、定型作業
  → Haiku 4.5（速い＆安い）
```

### モデル切り替え方法

```
/model          ← モデル選択メニューが表示される
/model opus     ← Opus 4.6に切り替え
/model sonnet   ← Sonnet 4.6に切り替え
/model haiku    ← Haiku 4.5に切り替え
```

## Sonnet 4.6の飛躍的進化

2026年2月にリリースされた**Sonnet 4.6は、ゲームチェンジャー**です。

### ベンチマーク比較

| ベンチマーク | Sonnet 4.6 | Opus 4.6 | 差 |
|-------------|-----------|----------|-----|
| **SWE-bench**（コーディング） | **79.6%** | 80.8% | わずか1.2pt差 |
| **OSWorld**（PC操作） | **72.5%** | 72.7% | ほぼ同等 |
| **数学・推論** | **89%** | — | 高スコア |

### Claude Codeでの実測結果

Anthropicの社内テストで、開発者がSonnet 4.6を選んだ割合：

- **vs Sonnet 4.5（前世代）**: **70%** がSonnet 4.6を選択
- **vs Opus 4.5（前フラッグシップ）**: **59%** がSonnet 4.6を選択

つまり、**前世代のフラッグシップモデルを超えた**。

### 何が良くなったのか

| 改善点 | 内容 |
|--------|------|
| **コンテキスト理解** | コードを修正する前に、既存のコードをしっかり読むようになった |
| **ロジックの統合** | 共通ロジックをコピペせず、適切に共通化してくれる |
| **過剰設計の抑制** | 不要な抽象化やオーバーエンジニアリングが大幅に減少 |
| **指示の遵守** | 指示通りに動く。勝手な判断で暴走しにくい |
| **正直さ** | 「できました」と嘘をつくことが減った。ハルシネーションの低減 |
| **マルチステップ** | 複数ステップのタスクを最後までやり切る力が向上 |

### コンテキストウィンドウ

Sonnet 4.6は**100万トークン**のコンテキストウィンドウ（ベータ）に対応。プロジェクト全体のコードベースを一度に把握できます。

### 結論：普段使いはSonnet 4.6で十分

Opusの**1/5のコスト**で、コーディング性能はほぼ同等。まずはSonnet 4.6をデフォルトにして、本当に複雑な設計タスクだけOpusに切り替えるのがベストプラクティスです。
